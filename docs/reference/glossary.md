# Glossary

<p class="lesson-lead">Definitions here are intentionally compact. Return to the corresponding course lesson when a term affects a decision you are making.</p>

## A–C

**Accessible name**  
The text or equivalent label an accessibility API exposes for an element, derived from sources such as visible text, `label`, `alt`, or ARIA naming attributes.

**Accessibility tree**  
A browser representation of meaningful roles, names, states, and relationships used by assistive technology.

**Attribute**  
Additional information written in an HTML start tag, such as `href`, `class`, `lang`, or `required`.

**Block box**  
A box that normally participates in block layout, stacking in the block direction and using available inline space.

**Box model**  
The content, padding, border, and margin layers used to calculate and separate element boxes.

**Breakpoint**  
A condition where a responsive design changes rules because content needs a different relationship—not a universal device category.

**Cascade**  
The CSS conflict-resolution process that chooses a winning declared value from relevant origins, importance, layers, specificity, scope, and order.

**Cascade layer**  
A named precedence group created with `@layer` that organizes how normal and important declarations compete.

**Combinator**  
A selector relationship such as descendant space, direct child `>`, next sibling `+`, or subsequent sibling `~`.

**Computed value**  
The value produced after the cascade, inheritance, and value processing, before or during later layout-dependent resolution as defined per property.

**Containing block**  
The rectangle an element uses to resolve certain sizes and positions, especially for positioned elements and percentages.

**Container query**  
A conditional CSS rule based on a query container's size or style rather than the entire viewport.

**Custom property**  
A cascaded value named with `--name` and commonly used through `var()` for tokens and component inputs.

## D–H

**Declaration**  
A CSS property-value pair such as `color: teal`.

**DOM (Document Object Model)**  
The browser's parsed, tree-shaped representation of an HTML document that scripts and developer tools can inspect and modify.

**Element**  
An HTML structure represented by tags and content, such as `<p>Text</p>`, or a void element such as `<img>`.

**Flexbox**  
A layout model for distributing and aligning items along one primary axis, with wrapping available when needed.

**Focus**  
The current element receiving keyboard or similar input. Visible focus lets users know where interaction will occur.

**Formatting context**  
The layout environment governing how boxes participate, such as block, flex, or grid formatting context.

**Grid**  
A two-dimensional layout model using rows, columns, tracks, lines, areas, and item placement.

**HTML**  
The markup language used to express web-document structure, meaning, relationships, and built-in controls.

## I–P

**Inheritance**  
The process by which some CSS properties use a parent's computed value when the child has no own winning declaration.

**Intrinsic size**  
A size contribution based on an element's content or natural media dimensions rather than only an external fixed constraint.

**Landmark**  
A meaningful page region such as navigation or main content that can support fast navigation in assistive technology.

**Logical property**  
A CSS property expressed in block/inline or start/end directions so it can adapt to writing mode and direction.

**Media query**  
A conditional CSS block based on media or viewport characteristics such as width or a user preference.

**Minimal reproduction**  
The smallest realistic code and conditions that still demonstrate a defect.

**Native element**  
An HTML element whose built-in semantics and behavior match the intended content or interaction.

**Normal flow**  
The default way block and inline content participates in document layout before flex, grid, floats, or positioning alter relationships.

**Progressive enhancement**  
A strategy that starts with useful content and baseline behavior, then adds presentation and advanced features without making the baseline unusable.

**Pseudo-class**  
A selector for state or structural condition, such as `:hover`, `:focus-visible`, or `:first-child`.

**Pseudo-element**  
A selector for a conceptual subpart or generated box, such as `::before` or `::first-line`.

## R–Z

**Replaced element**  
An element whose rendering comes from an external object or intrinsic representation, such as many images and form controls.

**Responsive design**  
An approach that lets content and layout adapt to available space, device characteristics, user preferences, and input conditions.

**Selector**  
A CSS pattern that queries the DOM to decide which elements a rule can style.

**Semantic HTML**  
HTML chosen for the meaning and behavior of content rather than its desired visual appearance.

**Specificity**  
A selector comparison used at one stage of the cascade after origin, importance, and layer have been considered.

**Stacking context**  
A local painting order whose children are layered together relative to other contexts.

**Token**  
A named design decision such as a color role, spacing step, radius, or type scale value, often implemented with a custom property.

**User agent**  
Software that retrieves and presents web content, most commonly a browser.

**Validation**  
Checking source against language rules to find structural or syntax problems. Validation is useful evidence but not a complete usability test.

**Viewport**  
The visible area used by the browser for page layout; its behavior on mobile is affected by viewport metadata.

**WCAG**  
The Web Content Accessibility Guidelines published by W3C, organized around perceivable, operable, understandable, and robust principles.

**Writing mode**  
The direction in which lines and blocks of text flow, affecting the meaning of logical inline/block properties.
