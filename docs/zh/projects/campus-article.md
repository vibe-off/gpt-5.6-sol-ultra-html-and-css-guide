# 实验 2：校园文章

<p class="lesson-lead">编辑型内容会检验信息层级、行长、图文、元数据、导航，以及在布局变化时保持阅读顺序不变的能力。</p>

<div class="lesson-meta">
  <span>项目</span>
  <span>2–3 小时</span>
  <span>响应式编辑布局</span>
</div>

## 任务说明

制作一篇题为《图书馆变成工作室的那一夜》的文章，其中包括：

- 网站页眉和主导航；
- 面包屑导航或清晰的返回路径；
- 文章标题、导语、作者和发布日期；
- 四个带标题的内容章节；
- 一幅图片及其图注；
- 一段在 HTML 中仍使用引用语义的重点引文；
- 相关活动列表；以及
- 页脚区域中的简报订阅表单。

请自行撰写真实可信的文案，或使用教师提供的文案。段落长度需要有所变化，且至少有一个标题长到会自然换行。

## 内容模型

```text
网站页眉
main
└─ article
   ├─ 文章页眉
   ├─ 正文段落
   ├─ 图片
   ├─ 引用
   └─ 文章页脚 / 标签
相关活动
网站页脚 / 简报订阅
```

判断相关活动应放在文章内部还是外部，依据是它们属于作者正文的一部分，还是页面级推荐内容。

## 视觉方向

- 温暖的阅读背景和高对比度文本；
- 正文宽度接近 `60–72ch`；
- 标题字号可流动变化，同时设有边界；
- 元数据清晰易读，不能小得难以辨认；
- 有一处更宽的媒体展示，但不会导致溢出；
- 相关内容只在空间允许时变为紧凑的侧栏区域；以及
- 明确设计打印行为：移除导航，同时保留文章内容。

## 布局约束

阅读顺序以常规流为基础。当相关区域移动到文章旁边时使用 Grid。不要用视觉重排让内容偏离 DOM 顺序。

```css
.article-layout {
  display: grid;
  gap: clamp(2rem, 5vw, 5rem);
}

@media (min-width: 64rem) {
  .article-layout {
    grid-template-columns: minmax(0, 1fr) minmax(14rem, 18rem);
    align-items: start;
  }
}
```

这段代码展示的是一种布局关系。请根据你的内容选择断点。

## 打印扩展

```css
@media print {
  .site-header,
  .site-footer,
  .related-events {
    display: none;
  }

  body {
    color: black;
    background: white;
  }

  a[href]::after {
    content: " (" attr(href) ")";
  }
}
```

评估直接打印 URL 对页面内片段、电子邮件链接或很长的目标地址是否有用。应根据实际情况完善，而不是盲目照抄。

## 证据任务

1. 展示标题层级大纲。
2. 测量文章在窄宽度和宽宽度下的计算行长。
3. 演示图片加载前，`figure` 如何预留宽高比空间。
4. 只用键盘完成所有导航和表单操作。
5. 预览一页打印效果，并记录你修改的两条规则。
6. 在 200% 缩放下测试最长的标题。

如果相关活动区域在 DOM 中位于文章之前，却在视觉上放在右侧，键盘顺序和阅读顺序会怎样？应该用 Grid 定位来“修正”内容顺序决策吗？</div>

## 与 AI 协作

请 AI 只从层级、行长、链接命名和打印后果四个方面评审阅读体验。不要让模型虚构用户研究，也不要让它声称某个行长在任何场景下都绝对完美。

## 现场变更

加入第二幅图片：图注有三行，源图片为竖向构图。文章在窄宽度和打印时都必须保持稳定。

<ul class="acceptance-list">
  <li>禁用 CSS 时，语义阅读顺序仍然正确。</li>
  <li>文章排版在窄屏、宽屏和缩放视图中都易于阅读。</li>
  <li>媒体具有符合用途的替代文本、图注、尺寸和流式大小。</li>
  <li>相关内容改变布局时，不改变逻辑顺序。</li>
  <li>键盘和打印检查都有记录。</li>
</ul>
