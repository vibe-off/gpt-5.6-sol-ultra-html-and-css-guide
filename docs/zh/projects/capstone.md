# 结课项目：校园信息中心

<p class="lesson-lead">为一个真实或虚构的校园组织制作小型网站。结果应当实用、稳健、可检查且易于修改，而不只是适合截图展示。</p>

<div class="lesson-meta">
  <span>结课项目</span>
  <span>8–12 小时</span>
  <span>构建 + 答辩</span>
</div>

## 选择一个组织

例如：

- 校园电影社；
- 学生研究成果展；
- 志愿辅导小组；
- 设计展览项目；
- 户外活动社团；或者
- 院系工作坊系列。

请使用真实可信的内容和需求。不要以会让人误认为官方网站的方式使用受商标保护的品牌。

## 必需页面

### 首页

- 清晰的组织身份和宗旨；
- 主导航；
- 精选活动或主要行动号召；
- 对三项活动的简短介绍；以及
- 联系或参与入口。

### 活动或资源

- 至少六个结构化条目；
- 有意义的标题、元数据和链接；
- 响应式集合布局；以及
- 在确有必要时使用一个数据表格或其他结构化信息。

### 加入或报名

- 清晰的说明；
- 姓名、联系方式和相关选项控件；
- 可见的标签与帮助文本；
- 原生输入类型和基础验证；
- 容易理解的错误与成功状态设计；以及
- 符合课堂原型场景、注重隐私的措辞。

无需收集或传输任何真实个人数据。项目可以保持为静态原型。

## 技术要求

### HTML

- 每个页面都有完整的文档外壳；
- 每页标题唯一，并只有一个清晰的 `h1`；
- 一致的地标区域和主导航；
- 当前页面指示；
- 有意义的链接文本和有效的相对路径；
- 符合用途的媒体替代文本和尺寸；以及
- 优先使用原生元素，再考虑自定义语义。

### CSS

- 使用一致的令牌表达核心视觉决策；
- 组件样式保持低特异性；
- 以常规流作为基础；
- 有目的地使用 Flexbox 和 Grid；
- 流式内容与由内容决定的断点；
- 可见焦点和完整的交互状态；
- 减少动态效果的行为；以及
- 不使用无法解释的 `!important` 或页面级溢出裁剪。

### 质量

- 浏览器控制台中没有无法解释的错误，资源加载也没有异常；
- 可以只用键盘完成导航和表单；
- 在 200% 缩放下仍可阅读；
- 从约 320px 到宽桌面都可用；
- 完成长内容测试；
- 针对一项现代特性做出当前适用的兼容性决策；以及
- 提供简明的证据和 AI 使用披露记录。

## 流程里程碑

1. **任务说明与内容清单** — 必须有什么？为谁服务？
2. **语义线框图** — 页面区域、标题大纲、链接、控件和数据。
3. **无样式 HTML 评审** — 内容顺序、语义、路径和表单关系。
4. **视觉系统** — 令牌、字体、间距、颜色和焦点。
5. **布局阶段** — 常规流、组件布局和响应式约束。
6. **质量阶段** — 键盘、缩放、长内容、动态效果、媒体和性能。
7. **现场变更** — 教师选择的需求变化。
8. **答辩** — 演示、浏览器证据、权衡和 AI 使用披露。

## 必需证据

- 每个页面的标题大纲；
- 一项层叠决策的计算样式追踪；
- 针对某项响应式修复的一次盒尺寸或布局测量；
- 键盘任务记录；
- 缩放和减少动态效果的测试记录；
- 一个兼容性来源和相应的回退决策；
- 一项前后对比的性能或传输观察；
- 一条已接受并验证的 AI 建议；
- 一条被拒绝的 AI 建议及其理由；以及
- 一个被缩减为最小案例的缺陷。

## 现场变更题库

第一个完整版本完成后，教师从中选择一项：

- 增加第七个导航目标，同时避免横向溢出。
- 将精选活动的标题和摘要长度增加一倍。
- 把活动卡片放进一个 300 像素宽的侧栏容器。
- 增加第二个表单选项，并提供条件式帮助文本。
- 使用现有角色令牌创建深色主题。
- 将主视觉图片替换为竖向素材，并保留有意义的裁剪。
- 更换强调色，同时保持对比度和状态区分。

学生有 25–40 分钟规划、实现、测试并解释这项变更。
