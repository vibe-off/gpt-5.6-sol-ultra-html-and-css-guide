# HTML 挑战：校园开放工作室

<p class="lesson-lead">先构建完整的内容结构，再编写自定义 CSS。移除样式表后，你提交的页面仍然应该容易理解。</p>

<div class="lesson-meta">
  <span>实践</span>
  <span>90–120 分钟</span>
  <span>仅使用 HTML</span>
</div>

## 任务说明

为校园开放工作室日创建一个页面。它需要包含：

- 网站标识与主导航；
- 一个页面标题与简短介绍；
- 三项工作坊摘要；
- 一张带说明文字的信息型图片；
- 以数据形式表达的日程；
- 一份报名表；以及
- 联系方式与场地信息。

你可以使用占位文案，但内容必须足够真实，以便做出语义选择。不要使用 `lorem ipsum`。

## 必需的内容关系

1. 每项工作坊都要有标题、描述、时间、地点与详情链接。
2. 日程表要有标题，以及清晰的行表头和列表头。
3. 表单收集姓名、邮箱、一项工作坊选择，以及可选的无障碍需求说明。
4. 每个控件都要有可见标签和用于提交的名称。
5. 图片替代文本与说明文字承担不同职责。
6. 场地地址应在适当位置做到机器可读，同时也要以文字显示。
7. 导航必须依据真实文件夹结构正常工作。

## 约束

- 使用完整的文档外壳。
- 原生元素已经适用时，不添加自定义 ARIA 角色。
- 不使用行内事件处理器。
- 不使用表格布局页面。
- 不仅为了视觉大小而选择标题级别。
- 不把 placeholder 作为唯一的表单标签。
- 使用一致的缩进，让源代码易读。

## 建议结构——不是答案

```text
body
├─ 跳过链接
├─ 网站页眉
│  └─ 主导航
├─ main
│  ├─ 页面介绍
│  ├─ 工作坊集合
│  ├─ 工作室 figure
│  ├─ 日程表格
│  └─ 报名表
└─ 网站页脚
```

请根据含义选择具体元素。树状图只是标注各部分的职责，并不要求每一行都变成 `section` 或 `div`。

## 需要提交的证据

提交页面，同时附上一份简短的 `evidence.md` 或课堂笔记，内容包括：

1. 标题大纲；
2. 一项标签/控件关联的截图或文字追踪；
3. 验证结果和你完成的修复；
4. 仅使用键盘完成操作的记录；
5. 一项你没有采用的语义决策及其原因；以及
6. 使用过的任何 AI 辅助、它的建议，以及你的验证方式。

## 验收标准

<ul class="acceptance-list">
  <li>文档结构有效且有明确意图，并有一个清晰的主要内容区域。</li>
  <li>标题层级与内容层级一致。</li>
  <li>链接与文件路径能够到达预期目标。</li>
  <li>图片替代文本反映图片用途，并且不重复说明文字。</li>
  <li>日程表保留表头关系。</li>
  <li>表单可用键盘操作，并为每个控件提供标签。</li>
  <li>没有自定义 CSS 时，页面仍然容易理解。</li>
</ul>

## 现场变更请求

首次评审后，在不重建整个文件的前提下完成以下修改：

> 新增第二个报名场次，并添加一则简短通知：摄影工作坊要求参与者自带相机。

解释你添加了哪些元素、哪些现有关系发生了变化，以及为什么表单仍然只提交一个含义明确的场次值。
