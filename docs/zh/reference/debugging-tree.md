# 调试决策树

<p class="lesson-lead">从症状出发，收集一项证据，再沿一个分支继续。证据尚未支持某个分支时，不要直接猜测具体属性。</p>

## 1. 能复现问题吗？

- 记录 URL/文件、视口、缩放、内容、输入方式和状态。
- 从源文件重新加载，不要只依赖 DevTools 中的临时修改。
- 如果问题不稳定，找出与时间、加载或交互有关的条件。

**不能：** 在打补丁前改进复现步骤。<br>
**能：** 继续。

## 2. 预期内容在 DOM 中吗？

**不在：** 检查 HTML 源码、解析结果、闭合标签、条件渲染、隐藏属性和控制台错误。验证最小片段。

**在：** 选中准确的元素并继续。

## 3. 所需资源加载了吗？

检查 Network 和 Sources 面板。

**没有：** 验证路径、基础 URL、文件名大小写、状态、内容类型和服务器行为。<br>
**有：** 继续。

## 4. 预期选择器匹配了吗？

**没有：** 将选择器与解析后的 DOM 对照。

- 类名拼写正确吗？
- 元素类型正确吗？
- 应该是直接子元素还是后代元素？
- 所需状态或属性存在吗？
- 媒体/容器条件生效了吗？
- 是否跨越了组件或作用域边界？

**匹配：** 继续。

## 5. 声明有效并处于活动状态吗？

**无效：** 检查拼写、值的语法、缺失的标点、不受支持的特性和变量替换。<br>
**被划掉：** 追踪来源、重要性、层叠层、特异性和源码顺序。<br>
**意外继承：** 找出最近一个拥有自身胜出值的元素。<br>
**活动：** 继续。

## 6. 计算值符合意图吗？

**不符合：** 展开计算属性追踪，找出胜出的声明。<br>
**符合：** 问题很可能出在后续的尺寸、布局、绘制或解释阶段。

## 7. 是尺寸或溢出问题吗？

同时检查元素及其包含块：

- 内容、内边距、边框、外边距；
- `box-sizing`；
- width/min/max 约束；
- 内容的内在尺寸；
- 图片尺寸；
- 很长且无法断行的内容；
- Flex/Grid 的自动最小值；
- 视口单位和滚动条；以及
- 变换后的视觉边界。

修复被违反的约束。不要默认使用页面级裁剪。

## 8. 是对齐或分布问题吗？

- 确认布局容器。
- 确认该项目是直接子元素。
- 确认主轴或 Grid 轨道。
- 测量可用的自由空间。
- 检查项目最小尺寸和自动外边距。
- 使用 Flex/Grid 叠加层。

如果没有自由空间，分布属性也无法凭空创造空间。

## 9. 是定位问题吗？

- 元素是否仍在常规流中？
- 它的包含块是什么？
- 哪些 inset 属性生效？
- 是否有祖先建立了滚动容器？
- `sticky` 是否有足够的滚动范围？
- 固定定位内容是否遮挡了有用的视口空间？

对于仍应影响周围布局的关系，请使用 Flexbox/Grid。

## 10. 元素被压在后面、被裁剪或不可见吗？

- 检查 `display`、`visibility`、透明度、裁剪和溢出。
- 检查祖先链上的层叠上下文。
- 比较祖先上下文，而不只是子元素的 `z-index` 值。
- 检查伪元素和覆盖层。
- 在实际主题中验证文本和背景颜色。

## 11. 是交互或无障碍问题吗？

- 原生元素选择合适吗？
- 它能获得焦点吗？
- 焦点可见吗？
- 键盘激活有效吗？
- 无障碍名称准确吗？
- 角色、状态和值是否同步？
- 隐藏内容是否也从焦点顺序中移除？
- 视觉顺序是否与 DOM 和焦点顺序一致？

仅添加 ARIA 并不能实现行为。

## 12. 补丁通过回归检查了吗？

重新测试：

- 原始状态；
- 邻近宽度；
- 短内容和长内容；
- 键盘和指针状态；
- 缩放；
- 如有深色主题或减少动态效果设置，也要测试；
- 共享同一规则的组件；以及
- 全新加载。

## 最小报告模板

```text
复现步骤：

实际结果：

```

如果报告无法说清证据，请回到最早存在不确定性的分支。
