# 术语表

<p class="lesson-lead">这里的定义刻意保持简短。当某个术语会影响你正在做的决策时，请回到课程中对应的完整章节。</p>

## A–C（英文索引）

**无障碍名称（Accessible name）**<br>
无障碍 API 为元素公开的文本或等效标签，可能来自可见文本、`label`、`alt` 或 ARIA 命名属性等来源。

**无障碍树（Accessibility tree）**<br>
浏览器对有意义的角色、名称、状态和关系所建立的表示形式，供辅助技术使用。

**属性（Attribute）**<br>
写在 HTML 开始标签中的附加信息，例如 `href`、`class`、`lang` 或 `required`。

**块盒（Block box）**<br>
通常参与块布局的盒子，会沿块方向依次排列，并使用可用的行内空间。

**盒模型（Box model）**<br>
用于计算和分隔元素盒子的内容、内边距、边框和外边距四个层次。

**断点（Breakpoint）**<br>
响应式设计因为内容需要新的排列关系而切换规则的条件，并不是一种通用设备分类。

**层叠（Cascade）**<br>
CSS 解决冲突的过程：从相关来源、重要性、层叠层、特异性、作用域和顺序中选出胜出的声明值。

**层叠层（Cascade layer）**<br>
通过 `@layer` 创建的具名优先级分组，用来组织普通声明和重要声明之间的竞争关系。

**组合器（Combinator）**<br>
表达选择器关系的符号，例如后代空格、直接子元素 `>`、下一个兄弟元素 `+` 或后续兄弟元素 `~`。

**计算值（Computed value）**<br>
经过层叠、继承和值处理后得到的值；根据具体属性，它位于后续依赖布局的解析之前或之中。

**包含块（Containing block）**<br>
元素解析某些尺寸和位置时所依据的矩形，尤其影响定位元素和百分比值。

**容器查询（Container query）**<br>
根据查询容器自身的尺寸或样式，而不是整个视口，决定是否应用 CSS 的条件规则。

**自定义属性（Custom property）**<br>
以 `--name` 命名、参与层叠的值，通常通过 `var()` 用作设计令牌或组件输入。

## D–H（英文索引）

**声明（Declaration）**<br>
一对 CSS 属性和值，例如 `color: teal`。

**DOM（文档对象模型，Document Object Model）**<br>
浏览器解析 HTML 文档后形成的树状表示，脚本和开发者工具可以检查和修改它。

**元素（Element）**<br>
由标签和内容表示的 HTML 结构，例如 `<p>文本</p>`；也可以是 `<img>` 这样的空元素。

**Flexbox**<br>
一种沿单个主轴分布和对齐项目的布局模型，并可在需要时换行。

**焦点（Focus）**<br>
当前接收键盘或类似输入的元素。可见焦点能让用户知道接下来会在哪里发生交互。

**格式化上下文（Formatting context）**<br>
决定盒子如何参与布局的环境，例如块、Flex 或 Grid 格式化上下文。

**Grid**<br>
一种使用行、列、轨道、网格线、区域和项目放置来构建二维布局的模型。

**HTML**<br>
用于表达网页文档结构、含义、关系和内置控件的标记语言。

## I–P（英文索引）

**继承（Inheritance）**<br>
当子元素没有自己的胜出声明时，某些 CSS 属性使用父元素计算值的过程。

**内在尺寸（Intrinsic size）**<br>
由元素内容或媒体的自然尺寸贡献的大小，而不是只由外部固定约束决定的大小。

**地标（Landmark）**<br>
导航或主要内容等有意义的页面区域，可帮助辅助技术快速跳转。

**逻辑属性（Logical property）**<br>
用块/行内或开始/结束方向表达的 CSS 属性，因此能适应书写模式和文字方向。

**媒体查询（Media query）**<br>
根据媒体或视口特征（例如宽度或用户偏好）有条件应用的 CSS 代码块。

**最小复现（Minimal reproduction）**<br>
仍能展示缺陷的最小且真实的代码与条件集合。

**原生元素（Native element）**<br>
其内置语义和行为与预期内容或交互相符的 HTML 元素。

**常规流（Normal flow）**<br>
在 Flex、Grid、浮动或定位改变元素关系之前，块级和行内内容参与文档布局的默认方式。

**渐进增强（Progressive enhancement）**<br>
先提供有用的内容和基础行为，再增加表现与高级特性，同时不让基础版本变得不可用的策略。

**伪类（Pseudo-class）**<br>
选择状态或结构条件的选择器，例如 `:hover`、`:focus-visible` 或 `:first-child`。

**伪元素（Pseudo-element）**<br>
选择概念性子部分或生成盒子的选择器，例如 `::before` 或 `::first-line`。

## R–Z（英文索引）

**替换元素（Replaced element）**<br>
其渲染来自外部对象或内在表示的元素，例如许多图片和表单控件。

**响应式设计（Responsive design）**<br>
让内容和布局适应可用空间、设备特征、用户偏好和输入条件的方法。

**选择器（Selector）**<br>
查询 DOM、决定某条规则可以设置哪些元素样式的 CSS 模式。

**语义化 HTML（Semantic HTML）**<br>
根据内容的含义和行为选择 HTML，而不是只根据期望的视觉外观选择。

**特异性（Specificity）**<br>
在来源、重要性和层叠层之后，层叠过程用于比较选择器的一项规则。

**层叠上下文（Stacking context）**<br>
一个局部绘制顺序，其中的子元素会作为整体，相对于其他上下文分层。

**令牌（Token）**<br>
具名的设计决策，例如颜色角色、间距级别、圆角或字号比例，通常通过自定义属性实现。

**用户代理（User agent）**<br>
获取并呈现网页内容的软件，最常见的是浏览器。

**验证（Validation）**<br>
依据语言规则检查源代码，以发现结构或语法问题。验证是有用的证据，但不是完整的可用性测试。

**视口（Viewport）**<br>
浏览器用于页面布局的可见区域；在移动设备上，它的行为会受到 viewport 元数据影响。

**WCAG**<br>
W3C 发布的《网页内容无障碍指南》，围绕可感知、可操作、可理解和稳健四项原则组织。

**书写模式（Writing mode）**<br>
文字行和内容块的流动方向，会影响逻辑属性中行内/块方向的含义。
