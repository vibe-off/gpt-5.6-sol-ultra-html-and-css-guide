# 八周课程地图

<p class="lesson-lead">默认路线兼顾概念、引导式练习、调试与结课项目。扩展或压缩课程时，请调整练习时间，而不是删掉心智模型。</p>

<div class="lesson-meta">
  <span>教师 + 学生地图</span>
  <span>8 周</span>
  <span>24–32 课时</span>
</div>

## 默认路线

| 周次 | 概念 | 实践课 | 学习证据 |
| --- | --- | --- | --- |
| 1 | 浏览器模型、文档结构、文本、链接 | 语义化校园文章 | DOM 检查 + 解释 |
| 2 | 语义、媒体、表格、表单 | 活动报名表 | 键盘与验证检查 |
| 3 | CSS 规则、选择器、层叠、继承 | 层叠问题现场 | 计算样式追踪 |
| 4 | 盒模型、排版、颜色、单位、常规流 | 个人资料卡 | 溢出与缩放测试 |
| 5 | 定位、Flexbox、Grid | 导航 + 课程仪表盘 | 布局选择答辩 |
| 6 | 响应式设计、状态、动效 | 响应式落地页 | 多宽度与减少动态效果检查 |
| 7 | 无障碍、调试、可维护 CSS | 修复 + 重构诊所 | 修改前后证据日志 |
| 8 | 现代 CSS、性能、结课项目 | 校园中心页 + 现场变更 | 演示、代码讲解、量规评审 |

## 四周冲刺课

短期模块仍须保留以下内容：

1. 语义化文档结构、链接、图片与表单；
2. 层叠、继承、盒模型与常规流；
3. Flexbox、Grid 与响应式约束；
4. 键盘/焦点基础与系统化调试循环；以及
5. 一个包含现场修改和解释的小项目。

动效、层叠层、容器查询、架构模式与更深入的性能工作可以作为扩展内容。不要用框架教程代替层叠或常规流课程，否则学生将缺少调试框架输出所必需的模型。

## 完整学期扩展

可以增加时间用于：

- 设计分析与内容层级；
- 更多表单状态与错误传达方式；
- 响应式图片与媒体格式；
- 打印样式和用户偏好样式；
- 浏览器兼容性与渐进增强；
- 设计令牌与组件清单；
- Git 协作与评审；
- 包含用户测试的多页面结课项目；以及
- 使用陌生代码反复开展调试诊所。

## 评估比例

| 证据 | 权重 | 它能揭示什么 |
| --- | ---: | --- |
| 预测与概念检查 | 10% | 使用工具辅助前的心智模型 |
| 渐进式实验 | 30% | 实现与验证习惯 |
| 调试实践考核 | 20% | 面对不确定性时依据证据诊断的能力 |
| 结课项目 | 30% | 综合质量与需求负责能力 |
| 现场修改 + 解释 | 10% | 学生能否修改作品并为之答辩 |

大型真实任务可以使用 AI，但必须披露。若目标是衡量学生自身的心智模型，简短的概念检查可以在不使用 AI 的情况下完成。

## 建议的每周节奏

1. **预测：**五分钟代码阅读题。
2. **建模：**简明讲解与现场检查。
3. **练习：**一项单独技能任务。
4. **暂停：**比较解决方案并说出取舍。
5. **构建：**把概念加入真实作品。
6. **验证：**依据明确的验收标准测试。
7. **解释：**完成课后检验题或简短讲解。

## 哪些内容应当推迟

不要在课程开始时就堆叠以下内容：

- 记忆大量罕见元素或属性；
- 大型构建系统；
- 掩盖布局决策的 CSS 框架；
- 不带内容或无障碍标准的像素级照抄；
- 与代码脱节的长篇提示词工程讲座；或
- 很快就会过时的兼容性表格。

教学生查阅最新资料，而不是假装一份固定清单永远有效。
