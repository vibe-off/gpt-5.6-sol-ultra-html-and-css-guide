# 学会浏览器原生的语言

<p class="lesson-lead">框架会变化，模型会变化。HTML、CSS、DOM 与浏览器渲染规则，始终是每个 Web 界面最终必须满足的底层标准。</p>

这门课程并不是标签与属性的快速浏览，而是对**实现、检查、调试与负责**的系统训练。你将使用纯 HTML 和 CSS 创建页面，让每项决策都清晰可见。你也会使用 AI，但只会在一套能检验其结论的流程中使用。

## 你将能够做到什么

完成课程后，你应该能够：

- 不依赖框架，构建含义清晰的 HTML 文档；
- 解释 DOM、层叠、盒模型、常规流、Flexbox 和 Grid 如何影响最终结果；
- 将一份小型视觉需求转换为响应式页面；
- 构建易用的导航、媒体内容、数据表格和表单；
- 在语义、键盘操作、焦点、标签、对比度和动效方面达到实用的无障碍基线；
- 根据浏览器证据诊断 HTML 和 CSS 问题，而不是随意修改数值；
- 用低优先级规则、可复用决策和清晰的组件边界组织 CSS；
- 审查 AI 生成代码的正确性、无障碍性与不必要的复杂度；并且
- 在需求现场发生变化时，修改并解释自己的作品。

## PAVE 循环

每个主要练习都遵循相同顺序：

1. **预测（Predict）**——说明你认为浏览器会怎样处理，以及原因是什么。
2. **提问（Ask）**——向 AI 或同伴请求聚焦的解释、评议或假设。
3. **验证（Verify）**——检查 DOM、计算样式、键盘行为和需求。
4. **解释（Explain）**——记录最终结果为何有效，以及有哪些证据支持它。

关键是从“代码看起来合理”走向“行为有证据支撑”。

## 课程如何组织

大多数课程包含：

- 一个明确的学习成果；
- 一个心智模型；
- 一个简短的完整示例；
- 一个运行前先预测的问题；
- 一项引导式练习；
- 一段 AI 评议或调试提示词；
- 一份浏览器验证清单；
- 常见失败模式；以及
- 一道要求你用自己的话解释概念的课后检验题。

各页面会通过用途自然形成分类：基础课程讲解概念，质量课程锻炼判断力，项目课程综合多个主题，参考资料则为正在进行的工作提供支持。

## 这门课程不要求什么

你不需要付费模型、云端 API、CSS 框架或高性能电脑。每项 AI 活动都可以改为与同伴讨论、分析教师提供的回答，或使用[调试决策树](/zh/reference/debugging-tree)。即使没有任何模型，练习本身仍然有价值。

::: tip 先建立基线
在第一课之前，不使用 AI 构建一个小页面，并把它保留下来。课程结束时，用同一个想法重新构建页面，比较结构、韧性与推理过程，而不只是外观。
:::
