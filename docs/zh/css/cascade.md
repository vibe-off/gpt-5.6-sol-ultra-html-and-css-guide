## 实用的层叠顺序

当多条声明都可能设置同一元素的同一属性时，依次问：

1. **相关性：**选择器匹配吗？媒体查询等条件成立吗？
2. **来源与重要性：**是浏览器、用户还是作者样式？是普通声明还是重要声明？
3. **层叠层：**如果声明位于不同层中，哪一层优先？
4. **选择器优先级：**哪个选择器权重更高？
5. **作用域邻近度：**使用 CSS 作用域时，哪个作用域距离元素更近？
6. **源码顺序：**如果前面的条件完全相同，后出现的声明胜出。

在初学者样式表中，你最常使用相关性、选择器优先级、继承和源码顺序。理解完整顺序，未来遇到层或重要规则时就不会觉得它们像“魔法”。

## 选择器优先级比较权重

```css
p { color: slategray; }              /* 类型选择器 */
.notice { color: teal; }             /* 类选择器 */
#deadline { color: navy; }           /* ID 选择器 */
```

在来源、重要性和层都相同的情况下，ID 选择器高于类选择器，类选择器高于类型选择器。无须把优先级当成十进制数；应依次比较 ID 列、类/属性/伪类列，再比较类型/伪元素列。

```css
.card .title { }      /* 两个类 */
.card-title { }       /* 一个类 */
```

第一条优先级更高，与嵌套结构的耦合也更强。应优先使用能够可靠表达组件职责的最低优先级。

## 继承不等于层叠

当元素没有自己的胜出声明时，`color`、`font-family` 等属性通常会从父元素继承。很多盒模型和布局属性不会继承。

```css
body {
  color: #182032;
  font-family: system-ui, sans-serif;
}
```

大多数后代都会得到这些文本值。如果链接有浏览器或作者定义的颜色规则，它自身的层叠值就会取代继承颜色。

常用的全局关键字：

- `inherit` — 采用父元素的计算值；
- `initial` — 采用该属性的初始值；
- `unset` — 属性通常继承时使用继承值，否则使用初始值；
- `revert` — 回退到更早来源或层的行为；
- `revert-layer` — 在层堆栈中回退。

## 用层明确组织优先顺序

```css
@layer reset, base, components, utilities;

@layer base {
  a { color: #9c321f; }
}

@layer components {
  .button { color: white; }
}

@layer utilities {
  .text-muted { color: #687083; }
}
```

对于普通声明，按此顺序排列时，后命名的层优先级更高。层可以明确架构上的优先关系，减少优先级争夺。未分层的普通作者样式高于已分层的普通作者样式，因此引入层时要有明确计划。

## `!important` 是诊断信号

重要声明会改变来源与层优先关系的计算。它可以用于范围狭窄且有文档说明的要求，但用它来“强制 CSS 生效”往往掩盖了选择器、加载顺序或架构问题。

添加之前：

1. 检查当前胜出的声明；
2. 找出它胜出的原因；
3. 降低不必要的优先级，或将规则移至正确的层；
4. 确认组件边界；
5. 如果仍必须使用重要声明，记录这个例外的理由。

## 在 DevTools 中追踪胜出声明

1. 选中准确的元素。
2. 在 Styles 面板中找到该属性。
3. 区分生效与被划掉的声明。
4. 打开 Computed 面板并展开属性追踪。
5. 记录选择器、源文件、行号和胜出原因。
6. 临时禁用一条声明来验证假设。
