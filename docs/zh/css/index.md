# CSS 是一套规则系统

<p class="lesson-lead">一条声明即使语法有效，也可能没有任何可见效果。分析 CSS 时，请依次问四个问题：文件加载了吗、选择器匹配了吗、声明胜出了吗、布局能够满足这个值吗？</p>

## 从外向内读懂一条规则

```css
.event-card > h2 {
  margin-block: 0;
  color: #273967;
}
```

- `.event-card > h2` 是**选择器**。
- 花括号中是**声明块**。
- `margin-block` 和 `color` 是**属性**。
- `0` 和 `#273967` 是**值**。
- 每组属性和值构成一条**声明**。

这个选择器表示“类名为 `event-card` 的元素所直接包含的 `h2`”。如果标题嵌套在另一层容器中，这条规则虽然有效，却不会匹配。

## 将 CSS 连接到 HTML

外部样式表可以让职责更清晰：

```html
<link rel="stylesheet" href="styles.css">
```

样式缺失时，应先确认请求成功，再修改选择器。路径错误、服务器响应失败，以及样式表在覆盖它的文件之前加载，是三个不同的问题。

## 四问调试阶梯

1. **加载了吗？** 样式表是否出现在 Network 和 Sources 面板中？
2. **匹配了吗？** 选中元素后，能否看到这条规则？
3. **胜出了吗？** 声明是生效状态还是被划掉了？它的值有效吗？
4. **得到满足了吗？** 父元素尺寸、显示模式、内容固有尺寸或其他布局约束是否影响了结果？

这套阶梯可以避免把“优先级”当作所有 CSS 错误的通用解释。

## 值具有类型，也依赖上下文

```css
.card {
  inline-size: min(100%, 42rem);
  padding: clamp(1rem, 3vw, 2rem);
  color: hsl(222 35% 23%);
}
```

值可以是关键字、长度、百分比、颜色、函数、图像，或该属性所接受的组合。百分比需要依据上下文换算；相对单位可能参照字号、视口或其他基准；`min()`、`clamp()` 等函数让你表达约束，而不只是一个固定数字。

如果某个值对该属性无效，浏览器会忽略这条声明，而不会停止处理整个样式表。

## 用类实现可复用样式

```html
<article class="event-card event-card--featured">…</article>
```

```css
.event-card { … }
.event-card--featured { … }
```

类可以创建可复用、优先级适中的样式钩子。元素选择器适合设置宽泛的默认样式。ID 通常更适合唯一的文档关系和片段目标，而不是日常组件样式。

这个元素可能不在类名为 `card` 的元素中，也可能不是 `h2`。类名不一致也是常见原因。

## 注释应该解释意图

```css
/* 在窄屏保留边距，同时维持适合阅读的行长。 */
.article {
  inline-size: min(100% - 2rem, 68ch);
  margin-inline: auto;
}
```

避免只把声明复述一遍的注释。应解释未来读者无法直接推断出的约束、变通方案或权衡。
